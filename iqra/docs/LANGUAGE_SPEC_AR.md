# مواصفات لغة IQRA

هذا المستند يصف قواعد اللغة الأساسية، الكلمات المفتاحية، أنواع البيانات، وبنية الجمل في لغة IQRA — مرجع لمستخدمي اللغة والمساهمين.

## نظرة عامة

IQRA هي لغة نصية (scripting) موجهة للتعليم والأتمتة، مع دعم للرموز العربية والإنجليزية.

ملاحظة: لغة العرض الافتراضية عربية. للتبديل إلى الإنجليزية في المخرجات، راجع `docs/OUTPUT_LANG_AR.md`.

## القواعد (Grammar)

- جملة تنتهي بـ `;` للفصل بين العبارات.
- التعابير الأساسية: الأعداد، السلاسل النصية، القيم البوليانية (`صحيح`/`خطأ`؛ تقبل أيضًا `true`/`false` في الشيفرة)، والقيمة الخالية `لاشيء` (يمكن كتابة `nil` في الشيفرة).
- التعابير الحسابية: `+`, `-`, `*`, `/`.
- المقارنات: `==`, `!=`, `<`, `>`, `<=`, `>=`.
- العمليات المنطقية: `&&`, `||`, `!` وكذلك كلمات منطقية بالعربية: `و`، `أو`، `ليس`.

## الكلمات المفتاحية (Keywords)

القائمة التالية تشمل الكلمات المفتاحية المدعومة (إنجليزية + عربية):

- تعليمية/تحكم: `if`, `else`, `while` — `اذا`, `وإلا`, `بينما`.
- التعريف/الوظائف: `let`, `fn`, `return` — `دع`, `دالة`, `رجع`.
- طباعات: `print` — `اطبع`.
- القيم: البولياني يطبع كـ `صحيح`/`خطأ`؛ وتظهر القيمة الخالية كـ `لاشيء`.
- عبارات منطقية: `and`, `or`, `not` — `و`, `أو`, `ليس`.

> ملاحظة: التعريف داخل المحلل (lexer) يدعم مطابقة الكلمات المفتاحية باللغتين؛ إن عثرت على اختلافات، افتح طلب تغيير.

## أنواع البيانات

- Number: أعداد قائمة على IEEE 754 (f64).
- String: سلاسل نصية مدعومة بالـ UTF-8.
- Bool: يكتب في الإخراج كـ `صحيح`/`خطأ`.
- Nil: تمثيل لغياب القيمة.
- List: قائمة مرتبة يمكن إنشاؤها عبر `list(...)`.
- Map/Dict: قاموس مفاتيح/قيمة عبر `قاموس(...)` أو `dict(...)` (كما يمكن استخدام `خريطة(...)` كمرادف عربي).
- Function: نوع دالة يتم تمريرها واستدعاؤها.

## بنية الجمل (Statements)

- العبارة التعبيرية: `expr;`
- التعيين: `x = 1;` أو `let x = 1;`.
- البلوكات: `{ ... }`.
- if/else:

```iqra
if cond { ... } else { ... }
```

- while:

```iqra
while cond { ... }
```

- تعريف دالة:

```iqra
fn name(args) { ... }
```

أو بالعربية:

```iqra
دالة اسم(أ, ب) { رجع أ + ب; }
```

## الدوال المدمجة (Builtins)

للاطلاع على قائمة كاملة مع الشرح، راجع `docs/BUILTINS_AR.md`.

## علامات الوقف العربية

- الممحلل اللغوي يتعرّف على الفاصلة العربية `،` والفاصلة المنقوطة العربية `؛`.
- يتعامل المحلل النحوي مع `،` كمُفصّل للعناصر/التعليمة و`؛` كمنهي للتعليمة.
- المزيد موجود في الكود المصدري ضمن `src/lang/runtime.rs` و`builtins`.

## قواعد التسمية

- المعرفات يمكن أن تبدأ بحرف (Unicode alphabetic) أو `_` وتستمر بأحرف أبجدية أو أرقام Unicode (بما في ذلك الأرقام العربية-الهندية) أو `_` أو حرف `؟` المستخدم في أسماء دوال مثل `رقم؟`.

## الأخطاء وتوجيه المستخدم

- أخطاء الlexer ترجع `Lex` مع موضع السطر والعمود.
- أخطاء وقت التشغيل ترجع رسائل مترجمة للعربية إن أمكن.

## أمان وتنفيذ النظام

- تنفيذ أوامر النظام محمي بطبقات متعددة:
  - منطقة ملفات محددة اختياريًا عبر `IQRA_FS_ROOT`: عمليات الملفات (`read_file`/`write_file`/`list_files`) تُرفض خارج الجذر المحدد برسالة "تم رفض الوصول".
    - تتم مقارنة المسارات بعد تطبيعها: استخدام `canonicalize` عند الإمكان، ومعه مسار بديل يعتمد على تفكيك المقاطع (بدون لمس نظام الملفات) عند عدم وجود الهدف.
    - تُمنع محاولات الاجتياز باستخدام `..` أو خلط الفواصل حتى لو لم يكن الملف موجودًا (تقليل مشاكل فحص/استعمال TOCTOU).
    - على ويندوز: المقارنة غير حساسة لحالة الأحرف وتُعامَل `\\` كـ `/`، كما يتم التعامل مع بادئة UNC `\\?\` بشفافية.
  - قائمة سماح للأوامر ومعايير منع: يمنع وجود محارف مثل `&`، `|`، `;`، `>`، `<` في الأوامر؛ وتُسمح فقط أوامر محددة حسب المنصة (مثل `echo`, `dir` على ويندوز).
  - مهلة تنفيذ الأوامر عبر `IQRA_SYSTEM_TIMEOUT_MS` (ميلي ثانية): عند تجاوز المهلة يتم إنهاء العملية وإرجاع مخرجات فارغة بدل خطأ لتعزيز أمان السكربتات.
  - الـ shell fallback اختياري فقط إذا تم تعيين `IQRA_ALLOW_SHELL_FALLBACK=1`.

- لاختبارات آمنة، قم بحقن منفذ نظام وهمي (mock) عبر `Runtime::new_with_executor`.

## ملاحظات حول الأداء

- يمكن إضافة اختبارات أداء باستخدام `criterion.rs`.

## اقتراحات مستقبلية

- توثيق كامل للـ AST، وشرح تحويل كل بنية لغة إلى عقد AST مفصلة.
- إضافة قسم عن نمط الأخطاء (error handling) والـ ABI للدوال المضمنة.
